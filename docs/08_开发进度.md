# 开发进度

## 1. 当前阶段

当前处于基础战斗原型向地图阶段过渡的阶段。核心实体已经可以表达玩家、敌人、炮弹、道具和基地；地图瓦片、文本加载和最小地图测试已经完成基础实现，但地图尚未接入游戏主流程。

## 2. 已完成内容

- CMake Presets 和 MSYS2 UCRT64 构建配置。
- SFML 3.1.0 FetchContent 集成。
- `Game` 主循环和基础窗口。
- 玩家坦克移动、转向、射击。
- 基础坦克生命、治疗、射击冷却。
- 轻型和重型敌方坦克属性区分。
- 炮弹移动、越界销毁、伤害和 ownerId。
- 基地受击和死亡状态。
- 血包和弹药强化包逻辑。
- 最小胜利和失败判定。
- `TileType`、`Tile` 和 `Map` 基础实现。
- `assets/maps/map_test.txt` 测试地图。
- `tests/map_test.cpp` 最小地图加载测试。

## 3. 部分完成内容

- 地图系统：可加载和查询文本地图，但尚未接入 `Game` 和碰撞系统。
- 敌人生成：当前固定生成两个敌人，没有生成器和关卡配置。
- AI：当前只有固定方向移动和自动射击。
- 碰撞：当前只处理矩形交叠，没有地形碰撞和坦克阻挡。
- 胜负流程：有状态判定，但没有菜单、重开或结算界面。

## 4. 待开发内容

- 地形和实体碰撞。
- 将 `Map` 接入 `Game`，用地图出生点替代硬编码位置。
- 玩家复活。
- 敌人生成管理。
- 简单、普通、困难 AI。
- HUD 和菜单。
- 地图编辑器。
- 局域网联机。
- 音效和动画。
- 可选强化学习 AI。

1. 修复和稳定现有实体基础代码。
2. 地图读取与绘制。
3. 地形和实体碰撞。
4. World 与关卡流程。
5. 玩家死亡、复活和胜负判定。
6. 敌人生成管理。
7. 轻型和重型坦克。
8. 简单 AI。
9. 血包和弹药强化包完整机制。
10. HUD 和菜单。
11. 普通、困难 AI。

## 6. 当前风险

- `Game` 类承担过多职责，后续扩展容易变复杂。
- 地图尚未接入游戏主流程，当前仍是独立模块。
- 除 `map_test.cpp` 外的测试文件为空，缺少完整回归保障。
- 配置文件未接入代码，参数仍主要写在代码常量中。

## 7. 近期任务

- 将地图加载接入 `Game`。
- 将地形碰撞接入玩家、敌人和炮弹。
- 补齐基础测试。
- 设计配置读取。
