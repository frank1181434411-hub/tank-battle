# 开发进度

## 1. 当前阶段

当前处于基础战斗原型阶段。核心实体已经可以表达玩家、敌人、炮弹、道具和基地，但地图、AI、联机、编辑器和资源系统仍未完成。

## 2. 已完成内容

- CMake Presets 和 MSYS2 UCRT64 构建配置。
- SFML 3.1.0 FetchContent 集成。
- `Game` 主循环和基础窗口。
- 玩家坦克移动、转向、射击。
- 基础坦克生命、治疗、射击冷却。
- 轻型和重型敌方坦克属性区分。
- 炮弹移动、越界销毁、伤害和 ownerId。
- 基地受击和死亡状态。
- 血包和弹药强化包逻辑。
- 最小胜利和失败判定。

## 3. 部分完成内容

- 敌人生成：当前固定生成两个敌人，没有生成器和关卡配置。
- AI：当前只有固定方向移动和自动射击。
- 碰撞：当前只处理矩形交叠，没有地形碰撞和坦克阻挡。
- 胜负流程：有状态判定，但没有菜单、重开或结算界面。

## 4. 待开发内容

- 地图读取与绘制。
- 地形和实体碰撞。
- 玩家复活。
- 敌人生成管理。
- 简单、普通、困难 AI。
- HUD 和菜单。
- 地图编辑器。
- 局域网联机。
- 音效和动画。
- 可选强化学习 AI。
- 自动化测试。

## 5. 推荐开发顺序

1. 修复和稳定现有实体基础代码。
2. 地图读取与绘制。
3. 地形和实体碰撞。
4. World 与关卡流程。
5. 玩家死亡、复活和胜负判定。
6. 敌人生成管理。
7. 轻型和重型坦克。
8. 简单 AI。
9. 血包和弹药强化包完整机制。
10. HUD 和菜单。
11. 普通、困难 AI。
12. 地图编辑器。
13. 局域网。
14. 音效和动画。
15. 可选强化学习 AI。

## 6. 当前风险

- `Game` 类承担过多职责，后续扩展容易变复杂。
- 地图和资源管理文件为空，后续需要尽早定义接口。
- 测试文件为空，缺少自动化回归保障。
- 配置文件未接入代码，参数仍主要写在代码常量中。

## 7. 近期任务

- 拆分 `World` 或关卡管理。
- 实现最小地图数据结构。
- 将地形碰撞接入玩家、敌人和炮弹。
- 补齐基础测试。
- 设计配置读取。
