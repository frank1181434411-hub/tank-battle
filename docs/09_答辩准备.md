# 答辩准备

## 1. 项目简介

Tank Battle 是一个基于 C++17 和 SFML 3.1.0 的复古坦克大战课程设计。当前版本实现了基础战斗原型，主要展示玩家坦克、敌方坦克、炮弹、道具、基地和最小胜负判定。

## 2. 演示流程

1. 使用 MSYS2 UCRT64 打开项目目录。
2. 执行 Debug 配置和构建命令。
3. 启动 `TankBattle.exe`。
4. 展示玩家移动和射击。
5. 展示敌人、炮弹、道具和基地。
6. 说明当前已完成和待完成模块。

## 3. 核心类说明

- `Game`：窗口、主循环、实体容器和状态判定。
- `Tank`：坦克公共基类。
- `PlayerTank`：玩家输入和道具效果。
- `EnemyTank`：轻型和重型敌人。
- `Bullet`：炮弹移动、伤害和归属。
- `Item`：血包和弹药强化包。
- `Base`：基地生命和绘制。

## 4. 关键流程说明

程序启动后创建 `Game`，初始化窗口、玩家、基地、敌人和道具。主循环中依次处理事件、更新实体和炮弹、执行碰撞判定、更新胜负状态并绘制画面。

## 5. 当前已完成功能

- 窗口创建和主循环。
- 玩家移动、转向和射击。
- 轻型和重型敌人。
- 炮弹飞行、越界销毁和伤害。
- 血包和弹药强化包。
- 基地受击。
- 最小胜负判定。

## 6. 创新点

当前可说明的设计点包括：

- 使用继承组织玩家坦克和敌方坦克。
- 使用 ownerId 区分炮弹归属。
- 将部分原型参数集中在 `prototype` 命名空间中。

## 7. 已知不足

- 地图模块已有基础实现，但尚未接入游戏主流程和地形碰撞。
- 地形碰撞尚未实现。
- AI 只有简单固定行为。
- 没有 HUD、菜单、音效和动画。
- 没有局域网联机。
- `map_test.cpp` 已用于地图最小加载测试，其他测试文件仍为空。

## 9. 常见问题

| 问题 | 回答要点 |
| -- | -- |
| 地图现在做到哪一步？ | 已有 `Tile`、`Map`、文本地图和最小加载测试，下一步是接入游戏主流程和碰撞 |
| 是否支持联机？ | 当前不支持，局域网是后续计划 |
| 是否有多难度 AI？ | 当前没有完整多难度 AI，只有固定移动和自动射击 |
| 为什么使用 SFML？ | SFML 提供窗口、图形、音频和网络基础能力，适合课程设计原型 |
