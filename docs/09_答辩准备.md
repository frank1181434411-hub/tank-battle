# 答辩准备

## 1. 项目简介

Tank Battle 是一个基于 C++17 和 SFML 3.1.0 的复古坦克大战课程设计。当前版本实现了基础战斗原型，并接入了文本地图、地形碰撞、砖墙破坏、坦克互撞、基地阻挡和草地覆盖绘制。AI 头文件和网络通信代码雏形已建立，但尚未完成 AI 行为实现和双端联机实机测试。

## 2. 演示流程

1. 使用 MSYS2 UCRT64 打开项目目录。
2. 执行 Debug 配置和构建命令。
3. 启动 `TankBattle.exe`。
4. 展示玩家移动和射击。
5. 展示敌人、炮弹、道具、基地和地图地形。
6. 展示砖墙破坏、地形阻挡、基地阻挡和草地覆盖效果。
7. 说明 AI 接口和网络通信代码当前只完成编写和构建接入，尚未完成运行验证。
8. 说明当前已完成和待完成模块。

## 3. 核心类说明

- `Game`：窗口、主循环、地图加载、实体容器、碰撞结算和状态判定。
- `Tank`：坦克公共基类。
- `PlayerTank`：玩家输入和道具效果。
- `EnemyTank`：轻型和重型敌人。
- `Bullet`：炮弹移动、伤害和归属。
- `Item`：血包和弹药强化包。
- `Base`：基地生命、绘制、受击和坦克移动阻挡。
- `Map` / `Tile`：文本地图加载、瓦片规则和分层绘制。
- `Collision`：地图阻挡、边界和炮弹命中瓦片查询。
- `AIController` / `EnemyAI`：已建立敌方 AI 接口和简单、普通、困难 AI 类声明。
- `NetworkSession`：已建立主机、客户端、输入和快照同步相关接口。
- `Hud`：计划新增的 UI 显示类，用于展示玩家血量、剩余敌人数和强化状态。

## 4. 关键流程说明

程序启动后创建 `Game`，加载测试地图，按地图出生点初始化玩家、基地和敌人，并创建临时固定道具。主循环中依次处理事件、更新实体和炮弹、执行地形与实体碰撞判定、更新胜负状态并分层绘制画面。

网络通信代码采用主机权威思路：客户端发送输入，主机更新游戏世界并发送快照，客户端应用快照。该流程已经写入代码并通过构建，但尚未完成双端实机测试。

## 5. 当前已完成功能

- 窗口创建和主循环。
- 玩家移动、转向和射击。
- 轻型和重型敌人。
- 炮弹飞行、越界销毁和伤害。
- 血包和弹药强化包。
- 基地受击。
- 最小胜负判定。
- 文本地图加载和游戏内绘制。
- 地形阻挡、炮弹撞墙和砖墙破坏。
- 玩家与敌人、敌人与敌人之间互相阻挡。
- 基地阻挡玩家和敌方坦克。
- 草地覆盖实体的分层绘制。
- AI 公共接口和敌方 AI 头文件。
- 网络通信代码雏形和构建接入。

## 6. 创新点

当前可说明的设计点包括：

- 使用继承组织玩家坦克和敌方坦克。
- 使用 ownerId 区分炮弹归属。
- 将部分原型参数集中在 `prototype` 命名空间中。
- 使用 `Map::drawLayer` 分离地图底层和草地上层。
- 使用 `boundsAt` 在坦克移动前预测下一步碰撞矩形。
- 使用 `TankCommand`、`AIContext` 和 `AIController` 分离敌人实体与 AI 决策。
- 网络通信采用主机权威快照同步思路，便于避免客户端各自结算导致状态分歧。

## 7. 已知不足

- 地图系统已接入测试地图，但还没有完整关卡流程和地图编辑器。
- AI 只有接口和头文件结构，实际游戏中仍是固定方向移动和自动射击。
- HUD、菜单、音效和动画尚未完成；HUD 已确定计划放入独立 `ui/` 模块。
- 道具随机刷新和生命周期尚未实现。
- 网络通信代码已编写并接入构建，但尚未完成双端实机连接和同步稳定性测试。
- `map_test.cpp` 已用于地图最小加载测试，其他测试文件仍为空。
