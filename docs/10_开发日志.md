## 2026-07-02 敌人初始化规则调整

### 调整背景

项目原设计使用通用敌人出生点 `E`，并考虑后续实现敌人生成管理、分批刷新或在场数量限制。

在进一步评估课程项目范围后，决定取消敌人波次、分批刷新和死亡后补充机制。关卡中的全部敌人改为由文本地图直接定义，并在地图加载完成后一次性创建。

### 调整原因

原生成方案需要额外维护：

- 敌人生成计时器；
- 待生成敌人数量；
- 最大同时在场数量；
- 出生点占用判断；
- 补充生成逻辑；
- 待生成敌人与当前敌人的联合胜利条件。

这些功能会增加游戏状态管理和网络同步的复杂度，并且不是当前课程项目的核心要求。

新方案让地图直接决定敌人的类型、数量和初始位置，规则更清晰，也更方便测试和答辩展示。

### 新地图标记

| 字符 | 含义 |
| --- | --- |
| `P` | 玩家初始位置 |
| `B` | 基地位置 |
| `L` | 轻型敌方坦克初始位置 |
| `H` | 重型敌方坦克初始位置 |

原通用敌人标记 `E` 停止使用。

### 新数据结构

地图模块增加敌人初始部署数据：

```cpp
struct EnemySpawn
{
    sf::Vector2f position;
    EnemyType type=EnemyType::Light;
};
```

`Map` 保存：

```cpp
std::vector<EnemySpawn> enemySpawns_;
```

地图中的每个 `L` 会生成一条 `EnemyType::Light` 部署数据，每个 `H` 会生成一条 `EnemyType::Heavy` 部署数据。

### 新初始化流程

1. 程序读取文本地图。
2. `Map` 解析地形字符和实体标记。
3. `P` 被记录为玩家初始位置。
4. `B` 被记录为基地位置。
5. 每个 `L` 被记录为轻型敌人初始部署。
6. 每个 `H` 被记录为重型敌人初始部署。
7. 所有实体标记对应的基础格子按空地处理。
8. `Game` 创建玩家和基地。
9. `Game` 遍历全部 `EnemySpawn`。
10. 系统一次性创建全部轻型和重型敌人。
11. 每辆敌人创建一个独立 AI 控制器。
12. 游戏进入正常主循环。

### 新胜利条件

敌人被消灭后从敌人容器中删除，不会复活，也不会补充生成新的敌人。

胜利条件为：

```cpp
enemies_.empty()
```

当全部初始敌人均被消灭时，游戏进入 `Victory` 状态。

由于不存在尚未生成的敌人，因此不需要额外判断待生成敌人数量。

### 坦克类型与 AI 难度分离

地图中的 `L` 和 `H` 只决定敌人的物理类型：

- 最大生命值；
- 移动速度；
- 炮弹伤害；
- 射击冷却；
- 外观和其他属性。

地图字符不决定敌人使用哪一种 AI。

`EasyAI`、`NormalAI` 和 `HardAI` 仍由游戏难度或关卡配置统一选择。这样可以让同一种坦克类型使用不同 AI 策略，避免物理属性与智能等级强制绑定。

### 删除或取消的设计

本次调整取消以下内容：

- 通用敌人出生点 `E`；
- 敌人波次系统；
- 分批刷新；
- 定时补充敌人；
- 最大同时在场数量；
- 敌人死亡后复活；
- 独立的敌人生成器或波次管理器计划。

地图编辑器仍不在当前项目范围内。
