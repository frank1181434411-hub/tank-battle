# 开发环境与构建

## 1. 开发环境

- Windows 10/11 64 位。
- MSYS2 安装目录：`D:\msys2`。
- 终端：MSYS2 UCRT64。
- 编译器：`D:\msys2\ucrt64\bin\g++.exe`。
- 调试器：`D:\msys2\ucrt64\bin\gdb.exe`。
- CMake：`D:\msys2\ucrt64\bin\cmake.exe`。
- Ninja：`D:\msys2\ucrt64\bin\ninja.exe`。
- C++ 标准：C++17。
- SFML：3.1.0。
- 编辑器：Visual Studio Code。

## 2. 依赖

项目通过 CMake `FetchContent` 获取 SFML 3.1.0，链接目标包括：

- `SFML::Graphics`
- `SFML::Audio`
- `SFML::Network`

当前没有其他已确认的第三方运行时依赖。

## 3. 项目配置

项目使用 `CMakePresets.json`。当前配置包含：

- 隐藏基础 Preset：`ucrt64-base`
- 配置 Preset：`debug`、`release`
- 构建 Preset：`debug`、`release`

`debug` 构建目录为 `build/debug`，`release` 构建目录为 `build/release`。

## 4. Debug 构建

```bash
cd /d/CPP/tank-battle
cmake --preset debug
cmake --build --preset debug
```

当前文档只记录命令，不在本文中声明新的构建测试结论。

## 5. Release 构建

```bash
cd /d/CPP/tank-battle
cmake --preset release
cmake --build --preset release
```

Release 构建用于发布前检查，当前没有专门的发布流程文档。

## 6. 运行方法

Debug 构建后执行：

```bash
./build/debug/bin/TankBattle.exe
```

当前可执行目标名称来自 `CMakeLists.txt` 中的 `TankBattle`。

## 7. VS Code 配置

`.vscode/settings.json` 已配置：

- 使用 CMake Presets。
- CMake 路径为 `D:/msys2/ucrt64/bin/cmake.exe`。
- C/C++ IntelliSense provider 为 CMake Tools。
- 默认终端为 MSYS2 UCRT64。
- UTF-8、LF、4 空格缩进。

`.vscode/launch.json` 已配置通过 CMake Tools 启动目标路径调试，并使用 `D:\msys2\ucrt64\bin\gdb.exe`。

## 8. 常见问题

| 问题 | 可能原因 | 处理建议 |
| -- | -- | -- |
| 找不到 `cmake` 或 `ninja` | 未进入 MSYS2 UCRT64 环境 | 使用 MSYS2 UCRT64 终端或 VS Code 默认终端 |
| SFML 下载失败 | 网络或 Git 访问问题 | 检查网络后重新运行 `cmake --preset debug` |
| 程序无法启动 | 构建未完成或运行路径错误 | 先执行 Debug 构建，再从项目根目录运行 |
| VS Code IntelliSense 异常 | 未配置 CMake preset | 先在 VS Code 中选择 `debug` preset |
