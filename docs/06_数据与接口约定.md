# 数据与接口约定

## 1. 基础枚举

当前存在于 `include/entity/entity.hpp`：

```cpp
enum class Direction { Up, Down, Left, Right };
enum class Faction { Player, Enemy };
```

当前存在于 `include/entity/enemy_tank.hpp`：

```cpp
enum class EnemyType { Light, Heavy };
```

当前存在于 `include/entity/item.hpp`：

```cpp
enum class ItemType { HealthPack, DamagePack };
```

当前存在于 `include/core/game.hpp`：

```cpp
enum class GameStatus { Running, Victory, Defeat };
```

## 2. 坦克数据

`Tank` 当前保存位置、方向、速度、生命值、阵营、当前炮弹伤害、射击冷却、炮弹速度、实体 ID 和 SFML 矩形形状。

公开接口包括：

- `update`
- `draw`
- `takeDamage`
- `heal`
- `isAlive`
- `getHp`
- `getMaxHp`
- `getId`
- `getBounds`
- `getPosition`
- `setPosition`
- `getDirection`
- `setDirection`
- `setHp`
- `boundsAt`

## 3. 炮弹数据

`Bullet` 当前保存：

- `position_`
- `velocity_`
- `damage_`
- `ownerId_`
- `faction_`
- `isAlive_`
- `shape_`

公开接口包括 `update`、`draw`、`isAlive`、`destroy`、`getDamage`、`getOwnerId`、`getFaction`、`getBounds`、`getPosition`、`getVelocity`。

## 4. 道具数据

`Item` 当前保存道具类型、存活状态和矩形形状。已有类型为血包和弹药强化包。

公开接口包括 `draw`、`isAlive`、`pickUp`、`getType`、`getBounds`。

## 5. 基地数据

`Base` 当前保存生命值和矩形形状。当前 `Game` 会把 `Base::getBounds()` 加入玩家和敌人的移动阻挡列表。

公开接口包括 `takeDamage`、`isAlive`、`getHp`、`setHp`、`draw`、`getBounds`。

## 6. 地图数据

当前已存在 `TileType`、`Tile` 和 `Map`。

`TileType` 包含：

```cpp
enum class TileType { Empty, Brick, Steel, Water, Grass };
```

`Tile` 当前提供：

- `type`
- `setType`
- `blocksTank`
- `blocksBullet`
- `isDestructible`
- `isDrawable`
- `drawLayer`
- `toChar`
- `fromChar`
- `color`

`Map` 当前保存地图宽高、瓦片数组、玩家出生点、基地位置、敌人出生点列表和加载错误信息。公开接口包括 `loadFromFile`、`loadFromLines`、`tileAt`、`setTile`、`tilePosition`、`tileCenter`、`draw`、`drawLayer` 等。

## 7. 碰撞接口

当前存在于 `include/world/collision.hpp`：

- `hitsTankBlock`
- `hitsBulletBlock`
- `hitsMapBounds`
- `findBulletHitTile`

这些接口使用 `sf::FloatRect` 与地图瓦片进行碰撞判断。当前用于玩家和敌人的地形阻挡、炮弹撞墙、砖墙破坏和炮弹越界删除。坦克之间和基地对坦克的阻挡使用 `sf::FloatRect::findIntersection` 判断。

## 8. 配置数据

状态：待设计。`config/game_config.ini` 存在但为空，当前代码未读取配置文件。

## 9. AI 输入和输出

状态：已建立接口，未完成实现。

当前存在：

- `TankCommand`：包含 `Direction direction`、`bool move`、`bool fire`。
- `AIContext`：包含地图、玩家列表、基地、敌人列表和炮弹列表的只读引用。
- `AIController::decide`：输入 `EnemyTank`、`AIContext` 和 `deltaTime`，输出 `TankCommand`。
- `EasyAI`、`NormalAI`、`HardAI`：已建立头文件声明，尚未完成 `.cpp` 实现。

当前敌人实体仍可通过 `setMoveDirection` 设置固定方向，并通过 `tryFire` 自动尝试射击。

## 10. 网络消息

状态：已建立接口，未完成双端实机测试。

当前存在于 `include/network/`：

- `NetworkMode`：`Offline`、`Host`、`Client`。
- `NetworkState`：`Disconnected`、`Connecting`、`Connected`、`Hosting`、`Failed`。
- `PlayerInputState`：记录移动和射击输入。
- `TankSnapshot`：记录坦克 ID、位置、方向、生命值和存活状态。
- `BulletSnapshot`：记录炮弹位置、阵营、伤害、拥有者、存活状态和速度方向。
- `ItemSnapshot`：记录道具位置、类型和存活状态。
- `GameSnapshot`：记录游戏状态、基地状态、玩家、敌人、炮弹、道具和已破坏砖墙列表。

当前 `NetworkSession` 支持主机监听、客户端连接、断开、更新、输入发送、快照发送、地块变化发送，以及对应消息轮询。实际局域网连接和同步稳定性尚未测试。

## 12. 地图文件格式

当前采用简单文本地图格式，已在 `assets/maps/map_test.txt` 中使用。

| 字符 | 含义 |
| -- | -- |
| `.` | 空地 |
| 空格 | 空地 |
| `#` | 砖墙 |
| `S` | 钢墙 |
| `W` | 水域 |
| `G` | 草地 |
| `P` | 玩家出生点，地形按空地处理 |
| `E` | 敌人出生点，地形按空地处理 |
| `B` | 基地位置，地形按空地处理 |
