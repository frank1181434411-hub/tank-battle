# 数据与接口约定

## 1. 基础枚举

当前存在于 `include/entity/entity.hpp`：

```cpp
enum class Direction { Up, Down, Left, Right };
enum class Faction { Player, Enemy };
```

当前存在于 `include/entity/enemy_tank.hpp`：

```cpp
enum class EnemyType { Light, Heavy };
```

当前存在于 `include/entity/item.hpp`：

```cpp
enum class ItemType { HealthPack, DamagePack };
```

当前存在于 `include/core/game.hpp`：

```cpp
enum class GameStatus { Running, Victory, Defeat };
```

## 2. 坦克数据

`Tank` 当前保存位置、方向、速度、生命值、阵营、当前炮弹伤害、射击冷却、炮弹速度、实体 ID 和 SFML 矩形形状。

公开接口包括：

- `update`
- `draw`
- `takeDamage`
- `heal`
- `isAlive`
- `getHp`
- `getMaxHp`
- `getId`
- `getBounds`

## 3. 炮弹数据

`Bullet` 当前保存：

- `position_`
- `velocity_`
- `damage_`
- `ownerId_`
- `faction_`
- `isAlive_`
- `shape_`

公开接口包括 `update`、`draw`、`isAlive`、`destroy`、`getDamage`、`getOwnerId`、`getFaction`、`getBounds`。

## 4. 道具数据

`Item` 当前保存道具类型、存活状态和矩形形状。已有类型为血包和弹药强化包。

公开接口包括 `draw`、`isAlive`、`pickUp`、`getType`、`getBounds`。

## 5. 基地数据

`Base` 当前保存生命值和矩形形状。

公开接口包括 `takeDamage`、`isAlive`、`draw`、`getBounds`。

## 6. 地图数据

当前已存在 `TileType`、`Tile` 和 `Map`。

`TileType` 包含：

```cpp
enum class TileType { Empty, Brick, Steel, Water, Grass };
```

`Tile` 当前提供：

- `type`
- `setType`
- `blocksTank`
- `blocksBullet`
- `isDestructible`
- `isDrawable`
- `drawLayer`
- `toChar`
- `fromChar`
- `color`

`Map` 当前保存地图宽高、瓦片数组、玩家出生点、基地位置、敌人出生点列表和加载错误信息。公开接口包括 `loadFromFile`、`loadFromLines`、`tileAt`、`setTile`、`tilePosition`、`tileCenter`、`draw` 等。

## 10. 地图文件格式

当前采用简单文本地图格式，已在 `assets/maps/map_test.txt` 中使用。

| 字符 | 含义 |
| -- | -- |
| `.` | 空地 |
| 空格 | 空地 |
| `#` | 砖墙 |
| `S` | 钢墙 |
| `W` | 水域 |
| `G` | 草地 |
| `P` | 玩家出生点，地形按空地处理 |
| `E` | 敌人出生点，地形按空地处理 |
| `B` | 基地位置，地形按空地处理 |
