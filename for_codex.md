# 一、总体设计目标

AI 必须与敌方坦克实体分离。

`EnemyTank` 负责：

* 位置。
* 移动。
* 方向。
* 生命值。
* 射击。
* 碰撞。
* 坦克类型参数。
* 执行 AI 给出的命令。

AI 控制器只负责：

* 选择方向。
* 决定是否移动。
* 决定是否开火。
* 选择目标。
* 后续进行寻路和躲避。

不要把随机移动、寻路、目标选择和炮弹躲避全部直接写进 `enemy_tank.cpp`。

坦克类型与 AI 难度必须分离。

坦克类型建议保留：

```cpp
enum class EnemyType
{
    Light,
    Heavy
};
```

AI 难度建议单独定义：

```cpp
enum class AIDifficulty
{
    Easy,
    Normal,
    Hard
};
```

例如：

```text
EnemyType::Heavy + AIDifficulty::Hard
```

表示使用重型坦克属性和困难 AI 行为。

不要创建：

```text
EasyTank
NormalTank
HardTank
AdvancedHeavyTank
```

等大量重复类。

# 二、推荐开发顺序

请先根据当前工作区确认以下流程是否适用，并指出需要调整的地方。

## 阶段 1：建立 AI 公共接口

建议新增：

```text
include/ai/
├── ai_controller.hpp
├── ai_context.hpp
└── tank_command.hpp
```

必要时可增加对应 `.cpp` 文件，但纯接口文件不必强制创建实现文件。

首先建立统一动作结构：

```cpp
struct TankCommand
{
    Direction direction;
    bool move;
    bool fire;
};
```

建立 AI 控制器基类：

```cpp
class AIController
{
public:
    virtual ~AIController()=default;

    virtual TankCommand decide(
        const EnemyTank& self,
        const AIContext& context,
        float deltaTime
    )=0;
};
```

建立只读世界信息：

```cpp
struct AIContext
{
    // 地图
    // 玩家状态
    // 基地位置
    // 其他坦克
    // 炮弹
};
```

具体字段必须根据当前项目实际接口决定，不得凭空编造。

不要直接把可修改的 `Game&` 或 `World&` 交给 AI。

## 阶段 2：补充 EnemyTank 执行动作接口

检查当前 `EnemyTank` 和 `Tank` 是否已经支持：

* 设置朝向。
* 开始移动。
* 停止移动。
* 尝试开火。
* 获取位置。
* 获取方向。
* 获取碰撞箱。
* 判断是否存活。
* 判断移动是否被阻挡。

如果当前 `setMoveDirection()` 同时完成转向和移动，建议评估是否需要拆分为：

```cpp
void setDirection(Direction direction);
void setMoving(bool moving);
```

或者保留现有接口并增加：

```cpp
void face(Direction direction);
```

不要为了 AI 大规模重写现有实体类。

## 阶段 3：实现简单 AI

建议新增：

```text
include/ai/easy_ai.hpp
src/ai/easy_ai.cpp
```

简单 AI 第一版只实现：

1. 随机选择方向。
2. 持续沿当前方向移动。
3. 每隔固定时间重新决策。
4. 遇到障碍物时改变方向。
5. 以一定概率开火。
6. 玩家或基地与敌人在同一直线且中间没有阻挡时，提高开火概率。
7. 尽量避免连续选择相反方向。
8. 不使用 BFS。
9. 不进行复杂炮弹躲避。

AI 决策不能每帧重新随机。

建议保存：

```cpp
float decisionTimer_;
float decisionInterval_;
TankCommand currentCommand_;
```

运行逻辑应类似：

```text
每帧：
更新计时器
如果还没到决策时间：
    返回上一次命令
如果到达决策时间：
    重新选择方向和射击行为
```

随机概率不能直接与帧数绑定。

不要使用：

```cpp
if(rand()%100<2)
```

并在每帧执行。

建议使用：

```cpp
std::mt19937
std::uniform_int_distribution
std::uniform_real_distribution
```

调试阶段应允许固定随机种子，以便复现问题。

## 阶段 4：移动受阻检测

AI 必须能知道坦克是否被墙体或其他坦克挡住。

请检查当前碰撞系统是否已经能够返回移动失败状态。

如果没有，可采用以下较小改动方案：

* AI 记录上一帧位置。
* 当前命令要求移动。
* 一段时间内位置几乎不变。
* 判定为卡住。
* 重新选择方向。

也可以在实体中增加：

```cpp
bool wasMovementBlocked() const;
```

但只有当前结构适合时才建议修改。

## 阶段 5：直线视线和射击判断

建议新增：

```text
include/ai/line_of_sight.hpp
src/ai/line_of_sight.cpp
```

或者先作为 `EasyAI` 的私有函数实现，后续再提取。

第一版只处理四方向射击。

判断：

* 玩家或基地是否与敌人近似处于同一横线。
* 玩家或基地是否与敌人近似处于同一竖线。
* 两者之间是否存在阻挡炮弹的地形。
* 射击路径上是否有友军。
* 当前坦克是否允许开火。

建议允许少量位置误差，不要求坐标完全相等。

## 阶段 6：接入游戏循环

游戏循环建议按以下顺序运行：

```text
构建 AIContext
→ AIController::decide()
→ 得到 TankCommand
→ EnemyTank 执行方向和移动命令
→ EnemyTank 尝试开火
→ 更新坦克和炮弹
→ 执行碰撞处理
→ 清理死亡实体
```

请根据当前项目实际游戏循环确认顺序，避免：

* AI 使用已经失效的实体引用。
* AI 在碰撞处理后重复移动。
* 同一帧执行两次射击。
* 死亡敌人继续决策。
* 暂停状态下继续更新 AI。

## 阶段 7：普通 AI

简单 AI 完成并稳定运行后，再增加：

```text
include/ai/normal_ai.hpp
src/ai/normal_ai.cpp
include/ai/pathfinder.hpp
src/ai/pathfinder.cpp
```

普通 AI 建议采用：

```text
有限状态机 + BFS 网格寻路 + 直线射击
```

状态可以先保持简单：

```cpp
enum class AIState
{
    Patrol,
    ChasePlayer,
    AttackPlayer,
    AttackBase,
    Stuck
};
```

BFS 只处理上下左右四个方向。

寻路必须使用地图现有的阻挡规则，不能重新写一套与碰撞系统不同的判断。

不要每帧重新寻路。

建议在以下情况重新规划：

* 目标位置发生明显变化。
* 路径为空。
* 当前路径被阻挡。
* 坦克被卡住。
* 地图中的砖墙被破坏。
* 距离上次规划超过固定时间。

## 阶段 8：困难 AI

普通 AI 稳定后，再增加困难 AI。

困难 AI只在普通 AI 基础上增加：

* 玩家和基地目标评分。
* 简单炮弹威胁检测。
* 简单横向或纵向闪避。
* 主动攻击挡路砖墙。
* 更短决策间隔。
* 更积极的射击判断。

不要重新实现另一整套坦克逻辑。

不要通过简单增加生命值或伤害来表示 AI 难度。
